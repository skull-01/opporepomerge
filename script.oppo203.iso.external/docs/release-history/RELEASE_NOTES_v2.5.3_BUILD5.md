# Release Notes — v2.5.3 Build 5

Build 5 adds a tester-facing hardware-validation readiness report export. The add-on can now generate a compact text report that records setup state, model/capability gates, Option 4 XML naming-test reminders, negative tests for loose/raw video, and the evidence fields a hardware tester should record.

This is not a playback-feature expansion and does not claim hardware validation. Builds 1–4 behavior remains preserved:

- Build 1: precise 4K UHD disc-style service interception.
- Build 2: conservative Option 4 XML routing.
- Build 3: version identity reconciliation.
- Build 4: `playercorefactory.xml` merge/backup/idempotency/rollback hardening.

Real OPPO/Chinoppo/Kodi/NAS/TV/ADB validation remains external and pending.
