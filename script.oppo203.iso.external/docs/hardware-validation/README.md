# Hardware Validation

Current public status:

```text
Software-verified release. Hardware validation not performed / not claimed.
```

The project includes software support paths, presets, and guarded helpers for several player, TV, and AVR families. That is not the same as real hardware validation.

## What counts as validation evidence

A useful hardware report should include:

- Add-on version and build identity
- Kodi version and platform
- Exact player model and firmware
- Connection method
- Media type and path style
- TV model and backend, if TV switching is enabled
- AVR model and backend, if AVR sequencing is enabled
- Expected and actual result
- Sanitized logs or screenshots if available

## Claiming support

Use exact language:

- `software-supported path`
- `tested by user on exact model/firmware`
- `not hardware validated`
- `experimental / warning-only`

Do not generalize one successful test to every clone, successor model, TV firmware, AVR model, or Kodi platform.
