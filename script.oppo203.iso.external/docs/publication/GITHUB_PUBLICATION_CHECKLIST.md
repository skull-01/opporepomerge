# GitHub Publication Checklist — v2.9.10 GitHub Ready

Use this checklist before making the repository public or creating a GitHub release.

## Required publication checks

- [ ] Repository is pushed from the `script.oppo203.iso.external-2.9.10-github-ready-dev-source.zip` source tree.
- [ ] Public root includes `README.md`, `LICENSE`, `CHANGELOG.md`, `CONTRIBUTING.md`, `SECURITY.md`, `SUPPORT.md`, and `CODE_OF_CONDUCT.md`.
- [ ] GitHub Actions workflow is present at `.github/workflows/ci.yml`.
- [ ] Dependabot configuration is present at `.github/dependabot.yml`.
- [ ] Issue templates and pull request template are present under `.github/`.
- [ ] Documentation is organized under `docs/` with user, developer, hardware-validation, publication, release-history, and AI-handoff sections.
- [ ] Historical reconstruction files are preserved under `docs/release-history/`, `docs/ai-handoff/`, and `docs/github-readiness/`.
- [ ] Runtime ZIP is generated by the allowlist packaging policy.
- [ ] Runtime ZIP excludes `tests/`, `tools/`, `scripts/`, `docs/`, `release-evidence/`, `.github/`, reports, and handoff files.
- [ ] SHA256 checksums are published with release assets.

## Truthfulness checks

- [ ] Release is described as software-verified.
- [ ] Hardware validation remains `not performed / not claimed` unless real tester evidence is added.
- [ ] No OPPO, Chinoppo, Magnetar, Reavon, Kodi/NAS, TV, ADB, Roku, LG, Samsung, Sony, Panasonic, Vizio, or AVR path is described as hardware-validated without evidence.
- [ ] Any hardware reports from users are stored separately and include model, firmware, connection method, media type, pass/fail result, and logs/evidence.
