# Support

For normal support requests, provide enough detail to reproduce the problem without exposing secrets.

## Useful information

- Add-on version and build identity
- Kodi version
- Operating system or device platform
- Media type: ISO, BDMV folder, MKV, MP4, or other
- Player model/profile selected in the add-on
- TV and AVR control options enabled or disabled
- NAS/path mapping approach, if relevant
- Sanitized Kodi logs
- Steps to reproduce

## Before filing a report

1. Confirm the same media path is reachable from Kodi.
2. Confirm the external player can reach the media location or mapped path.
3. Disable TV switching and AVR sequencing, then retest the base handoff.
4. Re-enable optional control paths one at a time.
5. Remove tokens, passwords, PSKs, and private network details from logs.

## Hardware validation reports

A successful report from one setup is valuable evidence for that setup, but it should not be generalized to all models or firmware versions. Use exact model and firmware details.
